## 二、项目验收目标

本项目的验收目标是完成一套能够实际运行的人脸识别考勤系统，并在普通人脸识别基础上加入红外温度活体检测与眨眼动作检测，提高系统对照片、普通热源等非真人对象的防护能力。

系统需要达到以下目标：

1. 能够通过摄像头实时采集人脸图像；
2. 能够完成用户注册和人脸特征保存；
3. 能够识别已注册用户；
4. 能够接收 MLX90640 输出的 32 × 24 红外温度矩阵；
5. 能够根据人脸区域判断对应红外 ROI 温度是否符合人体热区特征；
6. 能够通过眨眼动作判断是否为真实活体；
7. 能够在活体检测通过后自动写入考勤记录；
8. 能够在 Web 页面中查看用户和考勤结果；
9. 能够删除用户和考勤记录，便于测试和维护。

## 三、系统总体方案

系统由四个核心部分组成：

1. **昇腾开发板端**
   - 运行 Flask Web 服务；
   - 调用人脸检测和人脸识别模型；
   - 接收红外温度数据；
   - 完成人脸 ROI 温度判断；
   - 完成眨眼活体检测；
   - 写入并展示考勤记录。

2. **摄像头采集模块**
   - 使用 USB 摄像头采集实时视频；
   - 通过 `/video_feed` 接口在网页端显示；
   - 为人脸检测、注册抓拍和考勤识别提供图像来源。

3. **红外温度采集模块**
   - STM32F103 读取 MLX90640 红外热成像传感器；
   - 传感器输出 32 × 24 温度矩阵；
   - Windows 端脚本通过串口接收完整温度帧；
   - 将温度帧通过 HTTP 接口发送到昇腾开发板。

4. **Web 管理模块**
   - 用户管理页面；
   - 考勤页面；
   - 用户注册、删除；
   - 考勤记录查看、删除。

系统数据流程：

```text
USB 摄像头 → 昇腾开发板 → 人脸检测 → 人脸识别 → 注册用户匹配
MLX90640 → STM32F103 → Windows 串口脚本 → /api/temperature → 红外温度矩阵缓存
注册用户匹配 + 红外温度矩阵 → 人脸 ROI 温度判断 → 眨眼活体检测 → 写入考勤记录
```

## 六、创新点说明

### 6.1 多模态活体检测

本项目不是单纯依赖摄像头进行人脸识别，而是融合了可见光人脸识别、红外温度矩阵和眨眼动作检测三种信息。相比普通摄像头人脸识别系统，该方案能够更好地区分真实人体和照片、普通热源等非活体对象。

### 6.2 基于 32 × 24 红外矩阵的人脸 ROI 判断

MLX90640 的分辨率只有 32 × 24，不能直接获得高清热成像图。本项目根据摄像头检测到的人脸框，将人脸位置映射到红外矩阵中对应区域，再判断该区域温度是否符合人体热区特征。

### 6.3 人脸识别后再进行温度判断

系统没有对整张红外矩阵直接判断活体，而是在识别到人脸后，根据人脸 ROI 进行温度判断。这样可以避免环境中其他热源影响判断，提高红外温度判断的针对性。

### 6.4 眨眼动作作为最终活体条件

系统在红外温度判断通过后，还要求用户完成眨眼动作。这样可以避免单纯热源或静态照片通过系统，提高活体检测可靠性。

### 6.5 摄像头采集与识别处理分离

系统将摄像头采集线程和识别处理线程分离，避免模型推理阻塞视频采集，从而改善网页视频流卡顿问题。

### 6.6 支持考勤记录删除，便于测试

系统增加考勤记录删除功能，使同一用户可以反复测试注册、人脸识别、红外温度判断和眨眼检测流程，提升调试效率。

### 6.7 STM32 + 昇腾 + Web 的跨平台融合

本项目同时涉及 STM32 嵌入式传感器采集、Windows 串口数据转发、昇腾 AI 推理、Flask Web 服务、SQLite 数据库和前端页面交互，具有较强的综合性和工程实践价值。

## 十、后续优化建议

1. **红外与摄像头标定**：固定摄像头和 MLX90640 的安装位置，使用标定点生成 `thermal_calibration.json`。
2. **升级眨眼检测方法**：引入人脸关键点检测或 Eye Aspect Ratio，提高眨眼检测稳定性。
3. **增加随机动作挑战**：加入随机眨眼次数、转头、张嘴等动作，提高对视频攻击的防护能力。
4. **昇腾板直接读取串口**：将当前 STM32 → Windows → 昇腾板链路优化为 STM32 → 昇腾板串口直连。
5. **完善页面提示**：在考勤页面加入“请眨眼”“温度检测中”“考勤成功”等状态提示。
