# 昇腾人脸识别 + MLX90640 红外活体检测考勤系统使用说明

## 1. 项目简介

本项目仓库地址：

```text
https://github.com/zoujiafu-fufu/ascend-face-thermal-liveness
```

本项目是一套基于昇腾开发板、USB 摄像头、STM32F103 和 MLX90640 红外热成像传感器的人脸识别考勤系统。系统通过摄像头进行人脸检测与人脸识别，通过 MLX90640 获取 32 × 24 温度矩阵，并结合红外人脸 ROI 温度判断、手机矩形热源拦截、眨眼活体检测，实现较完整的活体考勤流程。

系统最终目标是防止以下情况误打卡：

- 使用照片进行人脸识别；
- 使用手机显示照片进行识别；
- 使用手机播放眨眼视频进行识别；
- 使用普通热源模拟人体温度；
- 使用移动照片或移动手机绕过简单活体检测。

系统最终判定逻辑为：

```text
人脸识别成功
AND
红外温度帧有效
AND
人脸 ROI 温度通过
AND
红外热源不像手机屏幕/矩形热源
AND
眨眼活体检测通过
AND
未处于重复考勤冷却时间
```

只有以上条件全部满足，系统才会写入考勤记录。

## 5. STM32F103 + MLX90640 工程使用

### 5.1 打开 STM32 工程

进入仓库中的：

```text
MLX90640/USER/
```

使用 Keil 打开：

```text
template.uvprojx
```

### 5.2 编译和下载

在 Keil 中完成以下操作：

1. 打开 `template.uvprojx`；
2. 确认芯片型号和实际 STM32F103 开发板一致；
3. 编译工程；
4. 通过 ST-Link 或其他下载器烧录到 STM32F103；
5. 连接 MLX90640；
6. 连接 USB 转串口模块到 Windows 电脑。

### 5.3 温度矩阵输出格式

STM32 端应输出 MLX90640 的完整温度矩阵：

```text
32 × 24 = 768 个温度点
```

推荐输出格式为每行 32 个浮点数，共 24 行。例如：

```text
35.77 34.92 36.05 ... 35.83
35.60 34.88 35.92 ... 35.40
...
```

Windows 端脚本累计 24 行后组成一帧完整温度矩阵，再发送到昇腾板。

## 10. 活体检测流程说明

完整考勤流程为：

```text
摄像头读取画面
    ↓
人脸检测
    ↓
人脸特征提取
    ↓
匹配注册用户
    ↓
读取最新红外温度矩阵
    ↓
检查温度帧是否异常
    ↓
将人脸 ROI 映射到 32 × 24 红外矩阵
    ↓
判断人脸 ROI 温度是否符合人体热区
    ↓
判断热区形状是否疑似手机屏幕/矩形热源
    ↓
如果不是手机热源，再进行眨眼检测
    ↓
眨眼通过后写入考勤记录
```

### 10.1 人脸 ROI 温度判断

系统根据摄像头检测到的人脸框，映射到红外矩阵中的对应区域，并计算：

- 环境温度 `ambient`；
- ROI 高位温度 `p90`；
- ROI 与环境的温差 `delta`；
- 热点数量 `hot_points`；
- 温度帧新鲜度 `age_ms`。

典型日志：

```text
Face ROI thermal: roi_live=True, map=rough_scale_no_calibration, mapped=(10,12), thermal_center=(14,17), ambient=26.62, p90=29.88, delta=3.26, hot_points=9, age=378ms, reason=roi liveness passed
```

### 10.2 手机矩形热源拦截

手机照片或手机视频可能通过人脸识别，因此系统在人脸 ROI 温度通过后、眨眼检测前，增加手机屏幕热源判断。

判断指标包括：

- 热区面积；
- 热区填充率；
- 热区宽高比；
- 热区温度标准差；
- 热区连通点数量；
- 热区外接矩形。

如果热源像手机屏幕或矩形热源，系统会直接拒绝，不进入眨眼检测。

手机被拦截时日志类似：

```text
Phone screen thermal blocked: score=0.85, fill=0.72, area=0.35, aspect=1.08, std=0.60, hot=112, bbox=(8,11)-(20,22)
User xxx matched, but thermal ROI liveness failed: blocked phone screen thermal source
```

### 10.3 眨眼检测

当且仅当以下条件通过后，系统才会进入眨眼检测：

```text
人脸识别通过
AND
人脸 ROI 温度通过
AND
不是手机矩形热源
```

眨眼通过日志：

```text
Action liveness passed: blink passed
```

## 11. 异常温度帧过滤

系统增加了温度帧合理性检查，防止串口错位或脏数据进入缓存。

异常示例：

```text
Thermal frame stored: min=24.79, avg=270.22, max=1222.77
```

这类温度明显不合理，系统会拒收。

过滤规则包括：

- 温度点数量必须足够；
- 有效点数不少于 700；
- 单点温度不能超过合理范围；
- 平均温度不能明显异常。

异常帧被拒绝时日志类似：

```text
thermal sanity rejected: temperature out of range
```

## 14. 常见问题与解决方法

### 14.1 网页打不开

检查 Flask 是否启动：

```bash
python3 app.py
```

确认输出包含：

```text
Running on http://10.148.111.242:5000
```

确认 Windows 和昇腾板在同一网络。

### 14.2 摄像头打不开

检查设备：

```bash
ls -l /dev/video*
```

测试摄像头：

```bash
python3 - <<'PY'
import cv2
for i in range(4):
    cap = cv2.VideoCapture(i, cv2.CAP_V4L2)
    ret, frame = cap.read()
    print(i, cap.isOpened(), ret, None if frame is None else frame.shape)
    cap.release()
PY
```

如果 `/dev/video2` 可用，确保权限：

```bash
sudo chmod 666 /dev/video2
```

### 14.3 数据库只读

如果出现：

```text
sqlite3.OperationalError: attempt to write a readonly database
```

检查数据库和目录权限：

```bash
ls -l attendance.db
ls -ld .
```

可以临时修复：

```bash
sudo chmod 666 attendance.db
sudo chmod 777 .
```

### 14.4 Windows 端发送温度失败

检查目标 IP 是否正确：

```text
http://10.148.111.242:5000/api/temperature
```

检查昇腾板 Flask 是否正在运行。

检查串口是否被其他软件占用。如果出现：

```text
PermissionError: 拒绝访问
```

关闭串口助手、Keil 串口监视器或其他占用 COM 口的软件。

### 14.5 温度发送端一直显示 `live=False`

这是正常现象。温度接口只负责保存温度帧，最终活体判断需要等摄像头识别到人脸后执行。

### 14.6 手机照片仍然进入眨眼检测

查看昇腾板日志中是否有：

```text
Phone screen check passed
```

如果手机照片没有被拦截，需要根据日志中的：

```text
score
fill
area
aspect
std
hot
bbox
```

继续调整手机热源拦截阈值。

### 14.7 模型文件缺失

如果出现模型加载失败，请确认模型文件已经放入：

```text
case1/models/
```

至少需要：

```text
face_detection.om
face_recognition.om
```

其他 ONNX 文件根据代码实际调用情况放置。

## 16. 后续优化建议

### 16.1 红外与摄像头标定

当前系统主要使用粗略比例映射：

```text
camera face ROI → thermal matrix ROI
```

后续建议固定摄像头和 MLX90640 的相对位置，并通过多个标定点生成：

```text
thermal_calibration.json
```

使用单应性矩阵提升红外 ROI 对齐精度。

### 16.2 升级眨眼检测

当前眨眼检测基于眼睛区域变化。后续可升级为：

- 人脸关键点检测；
- Eye Aspect Ratio；
- 随机眨眼次数；
- 随机转头；
- 张嘴动作；
- 多动作组合挑战。

### 16.3 昇腾板直接读取串口

当前温度链路为：

```text
STM32F103 → Windows 串口 → Python HTTP → 昇腾板
```

后续可以改成：

```text
STM32F103 → 昇腾板串口 → 本地解析温度矩阵
```

这样可以减少 Windows 中转，提高部署稳定性。

### 16.4 清理仓库文件

建议 GitHub 仓库只保留源码、模板、说明文件和必要配置。以下文件不建议上传：

- `.bak_*` 备份文件；
- `__pycache__/`；
- `.pyc` 文件；
- 数据库文件；
- 用户上传图片；
- Keil 编译输出；
- 超过 100MB 的模型文件。

## 17. 总结

本系统完成了从单一人脸识别到多模态活体检测考勤的升级。系统通过可见光摄像头识别人脸，通过 MLX90640 红外矩阵判断人体热区，通过手机矩形热源检测拦截手机照片和手机视频，并通过眨眼检测确认真实动作。

最终系统具备：

- 用户注册；
- 用户删除；
- 摄像头实时预览；
- 自动人脸识别；
- 红外温度矩阵接收；
- 人脸 ROI 温度判断；
- 手机屏幕热源拦截；
- 眨眼活体检测；
- 自动考勤；
- 考勤记录删除；
- 异常温度帧过滤。

该项目可以作为昇腾 AI 应用、红外传感器融合、嵌入式 STM32 数据采集和 Web 考勤系统的综合项目使用。
